📋 Cybersalt MCP for Joomla — Changelog

v1.5.0 2026-05-09

Feature release responding to the v1.4.x test feedback. Adds five new tools, total counts on paginated responses, and a token-substitute UI on the dashboard so non-technical users can copy a fully-ready prompt without manual editing. Tool count: 67 → 72.

📦New Features

  • get_plugin_params / set_plugin_params — generic plugin params read/write via direct DB. Unlocks site-wide schemaorg config (folder=system, element=schemaorg), router options, third-party plugin config, anything on any plugin's Options screen. Refuses protected/locked core plugins.
  • fetch_rendered_url(path, extract_jsonld?) — fetches a rendered page from the SAME Joomla site (same-origin only — no SSRF). Optional JSON-LD extractor pulls every <script type="application/ld+json"> block out of <head> and returns them parsed — closes the verification loop for Schema.org workflows.
  • set_article_custom_jsonld_bulk — bulk variant for sites with hundreds of articles. Per-item independent, capped at 500 per call.
  • get_4seo_config (4SEO add-on) — reads #__forseo_config where the real settings live (the Joomla extensions row's params field is near-empty for 4SEO).
  • total field on paginated responses — list_articles, list_articles_with_schema, list_users, and query_4seo_table now return the count across the whole filtered set so the agent knows when pagination is done without poll-till-empty.

🔧Improvements

  • Token-substitute UI on the dashboard's prompt tab. Optional input field for the user's token; when set, the Copy button substitutes the <PASTE YOUR JOOMLA API TOKEN HERE> placeholder before copying. Token never leaves the browser. Button label flips to "Copied with token included!" so the user knows substitution happened.

v1.4.2 2026-05-09

Patch release responding to a real-world test of the v1.4.x onboarding prompt — first-call friction fixes, no new tools.

🐛Bug Fixes

  • list_articles_with_schema summary was misleading. It counted only the rows on the current page. Fixed: summary runs COUNT(*) across the whole filtered set, and the response adds a top-level total so the agent knows the full filtered count.
  • Onboarding prompt: curl example mangled JSON arguments with nested quotes. Switched from inline -d to --data-binary @- with a single-quoted heredoc — survives any payload.
  • Onboarding prompt: tools/list and tools/call have different response shapes. Only the tools/call shape was documented. Added both.
  • Onboarding prompt: claude.ai branch was wrong. claude.ai connectors take URL + auth header in separate fields, not a JSON snippet. Fixed.
  • Onboarding prompt: prompt-injection guard. Added a "verify the URL matches your site's domain" check before Claude approves the claude mcp add dialog — defense against a maliciously edited prompt redirecting the install elsewhere.

v1.4.1 2026-05-09

🐛Bug Fixes

  • "Generate / view API token" link landed on the user list, not the admin's own profile. The link was task=user.edit with no id, so Joomla's controller fell through to the user list instead of opening the current admin's profile (where the API Token tab lives). Fixed by computing the current admin's user id and substituting it into task=user.edit&id={N}. Both link instances on the dashboard also now open in a new tab so the user keeps the dashboard open while copying the token.

v1.4.0 2026-05-09

Fixes broken tab switching on the dashboard, swaps the setup tab order so the easier path (copy-paste prompt) is the default, and adds a self-install offer to the prompt so Claude Code users can convert a one-off prompt into a permanent MCP connector with one follow-up sentence.

🐛Bug Fixes

  • Tabs not switching on dashboard. Bootstrap's bootstrap.tab JavaScript module isn't auto-loaded in Joomla admin views. Without an explicit WebAssetManager::useScript('bootstrap.tab') opt-in, clicking a tab only changed the URL hash; the panel never activated. Fixed.

🔧Improvements

  • Tab order swapped. The copy-paste prompt is now the default tab (renamed "Copy a prompt into Claude (recommended)") because it's the easier path for non-technical users. The JSON snippet path moves to the second tab ("MCP Connector (manual config)").
  • Self-installing prompt. The copy-paste prompt now includes a closing instruction telling Claude: after confirming the connection, offer to install this site as a permanent MCP connector via claude mcp add (Claude Code only). The user says "make it permanent" and Claude runs the install command (with Claude Code's standard approval dialog before any shell command runs). After restarting Claude, this site appears as native MCP tools in every conversation — no prompt needed ever again. One paste = either one-off use OR permanent install, user's choice via a single follow-up sentence.
  • Setup card framing rewritten so the recommended path is obvious and the bonus self-install behaviour is explained up front.

v1.3.0 2026-05-09

Adds a "Method 2" copy-paste prompt path to the dashboard so users can connect Claude to a site without ever touching client MCP config (best in Claude Code).

📦New Features

  • Two-method dashboard. Tabbed UI with two setup paths:
    • Method 1 — MCP Connector: copy a pre-filled JSON snippet, paste into your Claude client config (Claude Desktop / claude.ai connectors / claude mcp add), restart. One-time, persistent.
    • Method 2 — Copy-paste prompt: dashboard generates a complete instruction prompt baking in site URL, endpoint, auth, JSON-RPC shape, tool surface, workflow rules, and an example curl. Paste into a fresh Claude Code conversation; Claude talks to the MCP endpoint directly via curl. Best for one-off tasks or when you can't change client config.
  • Copy buttons with "Copied!" feedback on both setup payloads (clipboard API with text-selection fallback for older browsers).

🔧Improvements

  • Dashboard tools list and setup payloads are both auto-generated from the live tool registry — the prompt always reflects what's actually installed.

v1.2.0 2026-05-09

Adds the 4SEO add-on (10 tools wrapping the Weeblr com_forseo extension) and fixes a critical memory-exhaustion bug in dashboard load. Tool count: 57 → 67; domain count: 12 → 13.

📦New Features

  • 4SEO add-on (plg_system_csmcpforj4seo) — first paid-add-on-shaped sub-plugin. Bundled in the package for testing; structured so it can be split into its own SKU later.
  • Adds 10 tools: list_4seo_tables, describe_4seo_table, count_4seo_rows, get_4seo_component_info, get_4seo_component_params, set_4seo_component_params, query_4seo_table, insert_4seo_row, update_4seo_row, delete_4seo_row.
  • Works directly against #__forseo_* tables (4SEO has no public Web Services API). All write tools refuse any table not prefixed with forseo_; UPDATE / DELETE refuse if the WHERE clause would touch more than one row.

🐛Bug Fixes

  • Memory exhaustion on dashboard load (CRITICAL). RegisterToolsEvent extended Joomla\CMS\Event\AbstractEvent, whose argument processor walked argument values via reflection and recursed catastrophically once the registry argument carried 50+ tool instances each holding a DatabaseInterface reference. Switched to the simpler Joomla\Event\Event base class and refactored the dashboard to read tool metadata statically from the plugin classes (no event dispatch from the dashboard).

🔧Improvements

  • Server protocol identifier bumped to cs-mcp-for-j 1.2.0.

v1.1.0 2026-05-09

Adds a Schema.org / SEO tool domain wrapping Joomla's CORE plg_system_schemaorg system. Tool count: 51 → 57; domain count: 11 → 12.

📦New Features

  • Schema.org / SEO tool domain (6 tools): list_schema_types, list_articles_with_schema, get_article_schema, set_article_schema, set_article_custom_jsonld, clear_article_schema.
  • Supports all 9 CORE Schema types (Article, BlogPosting, Book, Custom, Event, JobPosting, Organization, Person, Recipe). Use Custom + set_article_custom_jsonld for FAQPage, Service, LocalBusiness, Product, Review, BreadcrumbList, or any other schema.org type Joomla does not ship a form for.
  • Writes go directly to #__schemaorg in the same shape Joomla's onContentAfterSave hook produces — the rendered JSON-LD blocks pick up changes on the next page load.

🔧Improvements

  • Server protocol identifier bumped to cs-mcp-for-j 1.1.0.

v1.0.0 2026-04-25

Initial release with 51 built-in tools across 11 Joomla admin domains.

📦New Features

  • MCP endpoint: Streamable-HTTP MCP server at /api/index.php/v1/mcp, authenticated by Joomla API token (X-Joomla-Token or Authorization: Bearer).
  • JSON-RPC 2.0 server: initialize, notifications/initialized, ping, tools/list, tools/call. Single messages and batches supported.
  • ACL gating: csmcpforj.use (read) and csmcpforj.write (write) actions. Super Users / Administrator / Manager pass automatically.
  • Tool registry: Plugins extend the surface by subscribing to the onCsMcpRegisterTools event.
  • Admin dashboard: Endpoint URL, ready-to-paste client config, permissions table, and grouped tool list.
  • Built-in tool set (51 tools): Articles, Categories (any extension), Tags, Menus, Users & Access, Modules, Extensions/Plugins, Templates, Languages, Custom Fields, System.

🔍Security

  • Per-tool permission gate runs before every tools/call execution.
  • access.xml ships with the component so per-action permissions appear in System → Permissions immediately after install.
  • All writes route through the corresponding com_* Administrator model so workflow, asset, and ACL side effects stay consistent.
  • Database access uses quoteName() and parameterised values throughout — no string concatenation into SQL.
  • delete_user refuses to delete the calling user.
  • set_extension_enabled refuses to touch protected or locked core extensions.
  • get_site_info deliberately omits secrets (DB password, mailer credentials, captcha keys).
